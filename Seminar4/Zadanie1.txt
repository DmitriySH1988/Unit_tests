Задание 1:

Использование тестовых заглушек (mocks) полезно при написании модульных тестов по нескольким причинам:

Они позволяют изолировать тестируемый код от зависимостей, таких как внешние сервисы, базы данных и т.д. Это позволяет тестировать конкретный модуль независимо от его внешних зависимостей.

Позволяют создать предсказуемую среду для тестирования, контролируя возвращаемые значения и поведение зависимостей.

Ускоряют процесс тестирования, так как заглушки могут возвращать заранее подготовленные значения без реальных вычислений или обращений к внешним ресурсам.


Если требуется проверить, что метод был вызван с определенными аргументами, то следует использовать "mock" с Mockito. В Mockito это можно сделать с помощью метода verify(), 
который проверяет, был ли вызван определенный метод с указанными аргументами.

Если требуется просто вернуть определенное значение или исключение в ответ на вызов метода, то следует использовать "stub" с Mockito. 
В Mockito это можно сделать с помощью метода when(), который позволяет задать ожидаемое поведение для метода.

Для имитации взаимодействия с внешним API или базой данных следует использовать "mock" с Mockito. 
При помощи Mockito можно создать заглушки для этих внешних зависимостей и контролировать их поведение в тестах.
